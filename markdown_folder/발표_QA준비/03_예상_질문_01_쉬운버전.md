## 1️⃣ 시스템 전체에 대한 질문

### Q1. 이 프로그램을 왜 만들었나요?

**쉬운 답변**:
회의록 쓰는 게 너무 힘들어서요!

**상세 답변**:
- 회의 1시간 하면 회의록 작성에 또 1시간 걸려요
- 말한 내용을 놓치기 쉬워요
- 나중에 "누가 뭐라고 했지?" 찾기 어려워요

**우리 프로그램을 사용하면**:
- 회의록 작성 시간: 1시간 → 4분
- 모든 말을 빠짐없이 기록
- 나중에 검색/질문 가능!

---

### Q2. 누가 사용하면 좋나요?

**쉬운 답변**:
회의를 자주 하는 사람은 누구나!

**예시 사용자**:
- 학생: 학급 회의, 동아리 회의, 조별 과제
- 회사원: 업무 회의, 미팅
- 선생님: 교직원 회의, 학부모 상담
- 동호회: 정기 모임, 행사 기획

**실제 사례**:
"학급 회장이 매주 회의록 쓰느라 힘들어했는데, 이제는 파일만 올리면 끝!"

## 2️⃣ 기술 구조에 대한 질문

### Q6. 프로그램은 어떻게 작동하나요?

**레고 블록에 비유**:

```
[사용자 화면 블록]
       ↕️
[프로그램 본체 블록]
       ↕️
[AI 작업 블록]    [데이터 저장 블록]
```

**쉬운 설명**:
1. 사용자가 웹사이트에서 파일 업로드
2. 프로그램이 파일을 받아서 AI에게 전달
3. AI가 음성을 글자로 변환하고 요약
4. 결과를 데이터베이스에 저장
5. 사용자 화면에 결과 표시

---

### Q7. 왜 웹 프로그램으로 만들었나요?

**쉬운 답변**:
앱 설치 없이 인터넷만 있으면 어디서든 사용할 수 있어요!

**장점**:
- **설치 불필요**: 웹사이트 주소만 입력하면 끝
- **자동 업데이트**: 새 기능이 추가되면 자동으로 적용
- **어디서나**: 컴퓨터, 태블릿, 핸드폰 모두 가능
- **공유 쉬움**: 링크만 보내면 친구도 볼 수 있음

**비교**:
- 앱: 다운로드 필요, 용량 차지, 업데이트 귀찮음
- 웹: 클릭만 하면 바로 사용!

---

### Q8. 데이터는 어떻게 저장하나요?

**쉬운 답변**:
두 개의 창고에 나눠서 저장해요!

**창고 1 (SQLite)**: 기본 정보 저장
- 회의 제목, 날짜, 시간
- 누가 올렸는지
- 파일 이름

**비유**: 도서관 대출 카드 (책 정보만 기록)

**창고 2 (ChromaDB)**: AI 검색용
- 실제 대화 내용
- 의미로 검색할 수 있게 특수 처리

**비유**: 책 내용을 전부 디지털로 저장 (검색 가능)

**왜 두 개를 쓰나요?**
- 창고 1: 빠르고 간단한 정보 (제목, 날짜)
- 창고 2: 복잡하지만 똑똑한 검색 (내용)

---

### Q9. 보안은 안전한가요?

**쉬운 답변**:
네, 매우 안전해요! 집에 자물쇠 여러 개 달듯이 여러 겹으로 보호해요.

**보안 장치**:
1. **로그인**: 구글 계정 (구글이 보안 관리)
2. **파일 암호화**: 파일 이름을 암호처럼 바꿈
3. **접근 제한**: 내 파일은 나만 볼 수 있음
4. **HTTPS**: 인터넷 전송 시 암호화 (도청 불가)

**예시**:
```
원본 파일: "1차회의.mp3"
저장 파일: "a3f7b2c1-4d5e-6f7g-8h9i-0j1k2l3m4n5o.mp3"
→ 다른 사람이 파일 이름을 봐도 뭔지 모름!
```

### Q10. 인터넷이 없어도 되나요?

**답변**:
아니요, 인터넷이 필요해요.

**왜 필요한가요?**
- AI가 인터넷 클라우드에 있어요
- 파일을 서버로 보내야 해요
- 결과를 받아와야 해요

**비유**:
- 넷플릭스처럼 인터넷이 있어야 영상을 볼 수 있는 것과 같아요!

**인터넷 속도**:
- 최소: 5Mbps (일반 가정 인터넷)
- 권장: 10Mbps 이상 (빠른 처리)

## 3️⃣ 성능에 대한 질문

### Q11. 처리 속도가 얼마나 걸리나요?

**쉬운 답변**:
30분 회의 녹음 파일을 올리면, 약 4분 후에 완성된 회의록을 받을 수 있어요!

**단계별 시간**:
- 파일 업로드: 10초 (인터넷 속도에 따라)
- 음성→글자 변환: 2-3분 (AI가 들으면서 작성)
- 요약 생성: 30초 (중요한 내용 추출)
- 마인드맵 생성: 20초 (구조화)

**비교**:
- 사람이 직접: 1시간 이상
- 우리 프로그램: 4분
- **15배 빠름!** ⚡

---

### Q12. 정확도는 어느 정도인가요?

**쉬운 답변**:
음성 인식 정확도는 95%예요! (100개 단어 중 95개 정확)

**테스트 결과**:
- 조용한 환경: 95-98% 정확
- 약간 시끄러움: 85-90% 정확
- 매우 시끄러움: 70-80% 정확

**더 정확하게 하려면**:
- 조용한 곳에서 녹음
- 마이크를 가까이
- 한 명씩 차례로 말하기
- 너무 빠르게 말하지 않기

**챗봇 정확도**:
- 20개 질문 테스트 → 20개 모두 정확히 답변 (100%!)

---

### Q13. 동시에 몇 명이 사용할 수 있나요?

**쉬운 답변**:
현재는 10-20명이 동시에 사용해도 문제없어요!

**비유**:
- 작은 식당: 10테이블 (현재)
- 확장 가능: 100테이블로 늘릴 수 있음

**서버 용량**:
- 현재: 작은 서버 (개발/테스트용)
- 실제 서비스 시: 큰 서버로 확장
- 자동 확장: 사용자 많아지면 자동으로 서버 추가

---

### Q14. 파일 크기 제한이 있나요?

**답변**:
현재는 16MB까지 가능해요.

**실제 의미**:
- 16MB = 약 30분 MP3 녹음 파일
- 또는 약 15분 고음질 M4A 파일

**더 큰 파일은?**
- 현재: 설정 변경으로 500MB까지 가능
- 향후: 제한 없이 (시간만 더 걸림)

**비교**:
- 카카오톡 파일 전송: 최대 300MB
- 우리 프로그램: 16MB (향후 확장 예정)

---

### Q15. 지원하는 파일 형식은?

**답변**:
5가지 파일 형식을 지원해요!

**오디오 파일**:
- ✅ MP3 (가장 흔함)
- ✅ WAV (고음질)
- ✅ M4A (아이폰 녹음)
- ✅ FLAC (최고 음질)

**비디오 파일**:
- ✅ MP4 (소리만 추출해서 사용)

**안 되는 파일**:
- ❌ 문서 (PDF, Word, 한글)
- ❌ 이미지 (JPG, PNG)
- ❌ 다른 동영상 (AVI, MKV)

## 4️⃣ 기능에 대한 질문

### Q16. 화자 구분은 어떻게 하나요?

**쉬운 답변**:
AI가 목소리를 듣고 "이건 A 목소리, 이건 B 목소리"로 구분해요!

**작동 원리**:
1. 음성의 높낮이, 속도, 특징을 분석
2. 비슷한 목소리끼리 묶음
3. 화자1, 화자2, 화자3... 번호 부여

**정확도**:
- 2-3명: 95% 이상 정확
- 4-5명: 85-90% 정확
- 6명 이상: 70-80% 정확 (섞일 수 있음)

**더 정확하게 하려면**:
- 목소리 톤이 확실히 다른 사람들
- 한 명씩 차례로 말하기
- 마이크 가까이에서 녹음

---

### Q17. 요약은 어떻게 만들어지나요?

**쉬운 답변**:
AI가 긴 대화를 읽고 중요한 내용만 뽑아서 짧게 정리해요!

**예시**:
```
원본 (500단어):
"그러니까 말이야, 저는 생각하기에는, 어... 그게 뭐냐면,
15일까지 완료하면 될 것 같은데, 뭐 그렇죠? 네네..."

요약 (20단어):
"프로젝트 완료 기한을 11월 15일로 제안"
```

**요약 기준**:
- 결정 사항 (○○하기로 함)
- 중요한 의견
- 액션 아이템 (누가 뭘 할지)
- 일정, 날짜, 숫자

---

### Q18. 마인드맵은 뭐예요?

**쉬운 답변**:
회의 내용을 나무 가지처럼 펼쳐서 보여주는 그림이에요!

**예시**:
```
        학급 축제
        /    |    \
    일정  예산  담당
     |     |     |
   11/15 20만원 철수
```

**장점**:
- 한눈에 전체 내용 파악
- 관계 이해 쉬움 (뭐가 뭐의 하위 항목인지)
- 시각적으로 기억하기 좋음

**누가 만들어요?**
AI가 자동으로 만들어줘요! 여러분은 그냥 보기만 하면 돼요.

---

### Q19. 챗봇은 어떻게 작동하나요?

**쉬운 답변**:
회의록을 다 읽고 있는 똑똑한 비서라고 생각하면 돼요!

**예시 대화**:
```
나: "다음 회의 일정은?"
챗봇: "11월 15일입니다." (회의록에서 찾아서 답변)

나: "누가 발표 담당이야?"
챗봇: "철수가 담당하기로 결정했습니다."
```

**작동 과정**:
1. 질문을 이해
2. 회의록에서 관련 내용 검색
3. 찾은 내용으로 답변 생성
4. 자연스러운 문장으로 답변

**장점**:
- 긴 회의록을 안 읽어도 됨
- 필요한 정보만 빠르게 찾음
- 자연스러운 대화 가능

---

- 관리자는 모든 회의록

### Q24. 수정이나 삭제는 어떻게 하나요?

**답변**:
회의록 화면에서 편집/삭제 버튼을 누르면 돼요!

**수정 가능한 것**:
- 제목
- 날짜
- 내용 (직접 수정 가능)

**삭제**:
- "삭제" 버튼 → 확인 → 완전 삭제
- 삭제하면 복구 불가능!
- 본인 또는 관리자만 삭제 가능
