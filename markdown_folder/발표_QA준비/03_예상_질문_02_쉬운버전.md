## 1️⃣ 음성 인식 (STT) 질문

### Q26. STT가 뭔가요?

**쉬운 답변**:
Speech To Text의 줄임말이에요. 말(Speech)을 글자(Text)로 바꾸는 거예요!

**예시**:
```
🗣️ "안녕하세요" (소리)
    ↓ (AI가 변환)
📝 "안녕하세요" (글자)
```

**실생활 예시**:
- 유튜브 자동 자막
- 시리, 빅스비 같은 음성 비서
- 카톡 음성 메시지 → 텍스트 변환

---

### Q27. 어떤 AI를 사용하나요?

**답변**:
구글이 만든 "Gemini 2.5 Pro"라는 AI를 사용해요!

**왜 Gemini를 선택했나요?**
- 한국어를 잘 알아들어요 (95% 정확도)
- 누가 말했는지도 구분해줘요
- 속도가 빨라요 (30분 → 3분)
- 무료로 사용할 수 있어요

**다른 AI와 비교**:
- 카카오 STT: 한국어 특화, 유료
- Whisper: 무료, 느림
- Gemini: 빠르고 정확, 무료 👑

---

### Q28. 음질이 안 좋아도 인식되나요?

**답변**:
어느 정도는 괜찮지만, 깨끗할수록 정확해요!

**음질별 정확도**:

| 음질 | 정확도 | 예시 |
|------|--------|------|
| 아주 좋음 | 95-98% | 조용한 방, 좋은 마이크 |
| 좋음 | 90-95% | 일반 회의실 |
| 보통 | 80-90% | 약간 시끄러운 곳 |
| 나쁨 | 60-80% | 식당, 카페 |
| 아주 나쁨 | 40-60% | 공사장 옆 |

**잘 인식하려면**:
- 조용한 곳에서 녹음
- 마이크를 가까이
- 에어컨, 선풍기 소음 줄이기
- 한 명씩 차례로 말하기

---

### Q29. 여러 명이 동시에 말하면 어떻게 되나요?

**답변**:
AI가 최대한 구분하려고 하지만, 섞일 수 있어요!

**상황별**:

**좋은 경우** ✅:
```
A: "저는 이렇게 생각합니다"
B: "네, 좋은 의견이네요"
A: "감사합니다"
```
→ AI가 정확히 구분!

**안 좋은 경우** ⚠️:
```
A: "저는—"
B: "아니 그게—"
A: "잠깐만—"
B: "들어봐—"
```
→ AI가 헷갈림!

**꿀팁**:
- 한 명이 말 끝나면 다음 사람이 말하기
- 손 들고 말하기 (회의 예절!)

---

### Q30. 사투리도 인식되나요?

**답변**:
표준어보다는 덜 정확하지만, 대부분 인식돼요!

**지역별**:
- 서울말: 95% ✅
- 경상도: 85-90% ⚠️
- 전라도: 85-90% ⚠️
- 제주도: 70-80% ⚠️

**이유**:
AI가 표준어로 학습했기 때문이에요.
하지만 문맥으로 추측해서 꽤 잘 맞춰요!

**개선 방법**:
- 조금 천천히 말하기
- 중요한 단어는 또박또박
- 극심한 사투리는 표준어로 순화

---

### Q31. 외국어는 안 되나요?

**답변**:
현재는 한국어만 지원해요!

**향후 계획**:
- Phase 2: 영어 추가
- Phase 3: 일본어, 중국어
- Phase 4: 다국어 자동 감지

**영어 회의는?**
- 현재: 한국어 회의만
- 영어 단어 일부는 인식 (프로젝트, 마케팅 등)
- 완전 영어 회의: 아직 불가

## 2️⃣ 화자 구분 질문

### Q32. 화자 구분은 어떻게 하나요?

**쉬운 답변**:
AI가 목소리의 특징을 보고 "이 목소리는 A, 저 목소리는 B"로 구분해요!

**구분 기준**:
- 목소리 높낮이 (남성/여성)
- 말하는 속도
- 목소리 톤
- 억양

**비유**:
친구들 목소리를 구분하는 것처럼, AI도 각 목소리를 기억해요!

---

### Q33. 화자가 몇 명까지 구분되나요?

**답변**:
이론상 무제한이지만, 현실적으로는 5-6명이 적당해요!

**인원별 정확도**:
- 2-3명: 95% 이상 ✅
- 4-5명: 85-90% ✅
- 6-7명: 70-80% ⚠️
- 8명 이상: 60% 이하 ❌

**많으면 왜 어려운가요?**
- 목소리가 비슷한 사람들
- 말 겹치는 경우 증가
- AI가 헷갈림

**해결 방법**:
- 회의는 5명 이하로
- 큰 회의는 파트별로 나눠서 녹음

---

### Q34. 같은 사람을 다른 사람으로 인식하는 경우는?

**답변**:
가끔 그럴 수 있어요. 특히 이런 경우:

**헷갈리는 상황**:
- 목소리 톤이 비슷한 사람들
- 한 사람이 감정에 따라 목소리 변화 (화내다가 진지해지면)
- 마이크 거리가 바뀌는 경우

**예시**:
```
A: "안녕하세요" (평상시 목소리)
   → AI: 화자1

A: "아니!!! 그건 아니죠!" (흥분한 목소리)
   → AI: 화자3으로 착각 ⚠️
```

**수정 방법**:
- 나중에 수동으로 수정 가능
- 대부분은 정확하니 걱정 안 해도 돼요!

---

### Q35. 남성/여성은 구분되나요?

**답변**:
네! 목소리 높낮이로 거의 정확히 구분돼요.

**정확도**:
- 남성 vs 여성: 98% 이상
- 성인 vs 어린이: 95% 이상

**예외**:
- 목소리 톤이 중성적인 경우
- 변성기 청소년

---

## 3️⃣ 대화 내용 처리 질문

### Q36. 청킹(Chunking)이 뭔가요?

**쉬운 답변**:
긴 대화를 작은 덩어리로 나누는 거예요!

**왜 나누나요?**
- 30분 회의 = 약 5000단어 (책 10페이지 분량)
- 한 번에 처리하면 너무 김
- 작게 나눠서 관리하면 편함

**비유**:
- 피자를 조각으로 나누는 것처럼!
- 한 판 통째로 먹기 어렵잖아요?

**나누는 기준**:
1. 화자가 바뀔 때
2. 시간이 오래 지났을 때 (1분 이상)
3. 글자가 너무 많을 때 (1000자)

---

### Q37. 대화에서 불필요한 말은 제거되나요?

**답변**:
네! "어...", "그러니까..." 같은 군더더기는 자동으로 정리돼요.

**제거되는 것**:
- "어...", "음...", "그...", "저..."
- "그러니까", "뭐", "이제"
- 의미 없는 반복

**예시**:
```
원본:
"어... 그러니까, 음, 저는, 뭐,
 15일까지 완료하면, 어, 좋을 것 같은데요?"

정리 후:
"15일까지 완료하면 좋을 것 같습니다"
```

---

### Q38. 요약은 누가 하나요?

**답변**:
Gemini AI가 자동으로 요약해줘요!

**요약 과정**:
1. 전체 대화 읽기
2. 중요한 내용 추출
3. 주제별로 정리
4. 간결한 문장으로 작성

**요약 기준**:
- **결정 사항**: ○○하기로 결정
- **액션 아이템**: 누가 뭘 할지
- **의견**: 중요한 발언
- **일정**: 날짜, 기한

**비유**:
선생님이 수업 내용을 칠판에 요약 정리하는 것처럼!

---

### Q39. 구어체를 문어체로 바꾸나요?

**답변**:
네! 말하는 투를 글쓰는 투로 자동 변환해요.

**예시**:

| 구어체 (말투) | 문어체 (글투) |
|--------------|--------------|
| "저는요, 생각하기엔요" | "저는 다음과 같이 생각합니다" |
| "뭐 그냥 15일이면 되지 않을까요?" | "15일까지 완료를 제안합니다" |
| "아니 그게 말이죠" | "해당 사항에 대해" |

**왜 바꾸나요?**
- 회의록은 공식 문서예요
- 나중에 읽기 편함
- 전문적으로 보임

---

### Q40. 중요한 내용을 놓칠 수도 있나요?

**답변**:
AI가 모든 말을 기록하니까 걱정 안 해도 돼요!

**안전 장치**:
- 원본 대화 내용 전부 저장
- 요약에서 빠졌어도 원본에서 찾을 수 있음
- 챗봇으로 검색 가능

**예시**:
```
요약: "프로젝트 기한 15일로 결정"

챗봇 질문: "왜 15일로 정했어?"
챗봇 답변: [원본 대화를 찾아서]
"15일이 명절 전이라 여유가 있다는
 철수의 의견이 받아들여졌습니다"
```

## 4️⃣ AI 생성 기능 질문

### Q41. 마인드맵은 어떻게 만들어지나요?

**답변**:
AI가 회의 내용을 읽고 주제별로 나무 구조를 만들어요!

**생성 과정**:
1. 요약문 분석
2. 주제와 하위 항목 파악
3. 계층 구조 만들기 (나무 가지처럼)
4. 마크다운 형식으로 저장
5. 화면에 예쁘게 표시

**예시 구조**:
```
학급 축제 (최상위)
├─ 일정 (주제 1)
│  ├─ 준비 기간: 2주
│  └─ 당일: 11/15
├─ 예산 (주제 2)
│  ├─ 총액: 20만원
│  └─ 항목별 배분
└─ 역할 분담 (주제 3)
   ├─ 기획: 철수
   └─ 진행: 영희
```

---

### Q42. 마인드맵 수정할 수 있나요?

현재는 AI가 자동 생성만 가능해요. 수동 수정은 향후 추가 예정!

**대안**:
- 다운로드해서 다른 프로그램에서 수정
- 회의록 내용을 수정하면 마인드맵도 다시 생성 가능

### Q44. 액션 아이템은 자동으로 추출되나요?

**답변**:
네! AI가 "누가 무엇을 할지" 자동으로 찾아줘요.

**추출 기준**:
- "○○가 ××를 하다"
- "○○에게 맡기다"
- "○○가 담당"

**예시**:
```
대화: "철수가 발표 자료 만들어주세요"

대화: "다들 의견 생각해와요"
     ↓
추출: ☑️ 전원: 의견 준비
```

### Q45. 회의 시간은 자동으로 계산되나요?

**답변**:
네! 처음부터 끝까지 시간을 자동으로 측정해요.

**자동 계산**:
- 총 회의 시간
- 화자별 발언 시간
- 각 안건별 소요 시간

**예시**:
```
총 회의 시간: 35분 23초

화자별 발언:
- 화자1: 15분 (42%)
- 화자2: 12분 (34%)
- 화자3: 8분 (24%)

안건별 시간:
- 일정 논의: 10분
- 예산 논의: 15분
- 역할 분담: 10분
```

## 5️⃣ 성능과 최적화 질문

### Q46. 처리 속도를 더 빠르게 할 수 있나요?

**답변**:
현재도 충분히 빠르지만, 더 빠르게 하려면:

**속도 향상 방법**:
1. **파일 압축**: 큰 파일을 압축해서 업로드
2. **좋은 인터넷**: 빠른 인터넷 사용
3. **서버 업그레이드**: 더 좋은 서버로 (비용 발생)

**현재 속도**:
- 30분 회의: 약 4분 처리
- 이미 사람보다 15배 빠름!

---

### Q47. 여러 파일을 한 번에 올릴 수 있나요?

**답변**:
현재는 한 번에 하나씩만 가능해요.

**이유**:
- 서버 부담 줄이기
- 각 파일을 정확하게 처리
- 오류 방지

**향후 계획**:
- 일괄 업로드 기능 추가
- 폴더째로 업로드
- 예약 처리 (밤에 자동 처리)

---

### Q48. 긴 회의(2시간 이상)도 가능한가요?

**답변**:
가능하지만 시간이 더 걸려요!

**처리 시간**:

| 회의 길이 | 처리 시간 |
|----------|----------|
| 30분 | 4분 |
| 1시간 | 7분 |
| 2시간 | 13분 |
| 3시간 | 20분 |

**권장 사항**:
- 2시간 넘으면 중간에 쉬는 시간에 나눠서 녹음
- 파트별로 나눠서 업로드

---

### Q49. 오류가 발생하면 어떻게 하나요?

**답변**:
대부분 자동으로 재시도하지만, 안 되면 이렇게 하세요:

**오류 종류별 해결**:

1. **업로드 실패**
   - 인터넷 연결 확인
   - 파일 크기 확인 (16MB 이하)
   - 파일 형식 확인 (mp3, wav 등)

2. **음성 인식 실패**
   - 음질 확인 (너무 시끄럽지 않은지)
   - 파일 재생 확인 (재생이 되는 파일인지)

3. **처리 시간 초과**
   - 파일이 너무 큰지 확인
   - 잠시 후 다시 시도

**고객 지원**:
- 메일로 문의
- 오류 메시지 캡처해서 보내기

---

### Q50. 동시 접속자가 많으면 느려지나요?

**답변**:
현재는 10-20명까지는 괜찮아요!

**서버 용량**:
- 현재: 작은 서버 (개발/테스트용)
- 향후: 자동 확장 서버

**비유**:
- 작은 식당: 10테이블 (현재)
- 손님 많아지면: 테이블 자동 추가! (클라우드)

## 🎯 핵심 요약

### AI가 하는 일 3가지
