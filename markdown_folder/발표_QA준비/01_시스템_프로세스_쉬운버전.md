# 회의록 AI 시스템 - 초보자를 위한 쉬운 설명

> 코드를 전혀 몰라도 이해할 수 있는 우리 시스템 설명서
> 작성일: 2025-11-11

## 🎯 우리가 만든 프로그램은 뭐하는 거예요?

**간단히 말하면**: 회의 녹음 파일을 업로드하면, AI가 자동으로 회의록을 만들어주는 프로그램이에요!

**예시로 설명하면**:
1. 학급회의를 녹음했어요 (30분짜리 음성 파일)
2. 우리 프로그램에 업로드해요
3. 기다리면... 짜잔!
   - 누가 뭐라고 말했는지 정리된 문서
   - 중요한 내용 요약
   - 대화 내용을 시각적으로 보여주는 마인드맵
   - 궁금한 거 물어볼 수 있는 챗봇

모두 자동으로 만들어져요! 🎉

---

## 📱 사용자 입장에서 보는 과정

### 1단계: 로그인
- 구글 계정으로 로그인해요 (Gmail 있으면 됨!)
- 안전하게 내 정보를 보호해줘요

### 2단계: 파일 업로드
**지원하는 파일**:
- 녹음 파일: mp3, wav, m4a, flac
- 동영상 파일: mp4 (소리만 추출해서 사용)

**입력하는 정보**:
- 회의 제목 (예: "1차 학급회의")
- 회의 날짜 (예: 2025년 11월 10일)
- 녹음 파일

### 3단계: 기다리기 (AI가 일하는 시간!)
진행 상황이 화면에 표시돼요:
```
[20%] 음성을 글자로 바꾸는 중...
[40%] 누가 말했는지 구분하는 중...
[60%] 중요한 내용 요약하는 중...
[80%] 마인드맵 만드는 중...
[100%] 완료!
```

### 4단계: 결과 확인
생성된 자료들:
- **회의록**: 정리된 문서 (인쇄 가능)
- **대화 내용**: 누가 언제 뭐라고 말했는지
- **요약**: 중요한 내용만 간추림
- **마인드맵**: 회의 내용을 나무 구조로 시각화
- **챗봇**: "결정된 사항은 뭐야?" 같은 질문 가능

## 🤖 AI가 하는 일 (뒤에서 벌어지는 일)

### 1. 음성을 글자로 바꾸기 (STT)

**쉽게 말하면**:
유튜브 자동 자막처럼, 말하는 소리를 글자로 바꿔줘요.

**우리 AI의 특별한 점**:
- 누가 말했는지도 구분해줘요 (화자1, 화자2, 화자3...)
- 언제 말했는지 시간도 기록해요 (00:05:23에 화자1이 말함)

**예시**:
```
음성: "안녕하세요. 회의를 시작하겠습니다."

AI가 만든 글:
[화자1, 00:00:00] 안녕하세요. 회의를 시작하겠습니다.
```

### 2. 대화 내용 나누기 (청킹)

**왜 나눠요?**
- 30분 회의 = 약 5000자 (A4용지 3장 분량)
- 한 번에 읽으면 너무 길어요!
- 작은 덩어리로 나눠서 관리해요

**어떻게 나눠요?**
- 화자가 바뀔 때
- 시간이 오래 지났을 때 (1분 이상 침묵)
- 글자가 너무 많을 때 (1000자 넘으면)

**예시**:
```
청크1: [화자1] "안녕하세요. 오늘 안건은..."
청크2: [화자2] "네, 저는 다음과 같이 생각합니다..."
청크3: [화자1] "좋은 의견이네요..."
```

### 3. 요약 만들기

**쉽게 말하면**:
30분짜리 대화를 2-3분 만에 읽을 수 있게 요약해줘요.

**요약 방식**:
- 주제별로 정리: "프로젝트 일정", "예산 논의", "다음 회의"
- 중요한 말만 추출
- 구어체(말투) → 문어체(글)로 변환

**예시**:
```
원본: "그러니까, 뭐, 저는 있잖아요,
      15일까지 완료하면 좋을 것 같은데요, 어?"

요약: 프로젝트 완료 기한을 11월 15일로 제안함
```

### 4. 마인드맵 생성

**쉽게 말하면**:
회의 내용을 나무 가지처럼 펼쳐서 보여줘요.

**구조**:
```
        1차 학급회의
        /    |    \
   일정   예산   담당자
    |      |      |
  15일   10만원  철수, 영희
```

### 5. 챗봇 (질문하면 답해주는 AI)

**할 수 있는 일**:
- "결정된 사항은 뭐야?" → AI가 회의록에서 찾아서 답변
- "다음 회의는 언제야?" → 날짜 알려줌
- "철수가 한 말은?" → 철수 발언만 모아서 보여줌

**작동 원리**:
1. 질문을 AI가 이해해요
2. 회의록에서 관련된 내용을 찾아요
3. 답변을 만들어줘요

## 🏗️ 시스템 구조 (건물에 비유)

우리 프로그램은 3층 건물이라고 생각하면 돼요:

### 1층: 사용자가 보는 화면 (프론트엔드)
- 로그인 화면
- 파일 업로드 화면
- 결과 보는 화면
- 챗봇 대화창

**사용 기술**: HTML, CSS, JavaScript (웹사이트 만드는 언어)

### 2층: 프로그램 로직 (백엔드)
- 파일 받기
- AI한테 일 시키기
- 결과 저장하기
- 사용자 권한 체크

**사용 기술**: Python + Flask (프로그램 언어)

### 3층: 데이터 저장소 (데이터베이스)
- **SQLite**: 회의 정보, 사용자 정보 (표처럼 정리)
- **ChromaDB**: AI가 검색할 수 있게 특별한 형태로 저장

## 💡 실제 사용 예시

### 시나리오: 학급 임원 회의

**1. 회의 전**
- 녹음기 준비 (핸드폰 녹음 앱 사용)
- 회의 시작 버튼 누르고 녹음 시작

**2. 회의 중**
- 자유롭게 회의 진행
- 녹음만 하면 끝!

**3. 회의 후**
- 녹음 파일을 우리 프로그램에 업로드
- 4분 정도 기다리면 자동으로 회의록 완성
- 친구들에게 공유 (링크 복사해서 카톡으로 전송)

**4. 나중에**
- "저번에 결정한 게 뭐였지?" → 챗봇에게 물어보기
- "철수가 뭐라고 했더라?" → 검색해서 찾기

## ❓ 자주 묻는 질문

**Q: 인터넷 없어도 돼요?**
A: 아니요, 인터넷 필요해요. AI가 클라우드에서 작동하거든요.

**Q: 비용이 드나요?**
A: 현재는 무료예요! (나중에는 유료 옵션 생길 수도)

**Q: 파일 크기 제한 있어요?**
A: 현재는 16MB까지 (약 30분 녹음 파일)

**Q: 개인정보는 안전한가요?**
A: 네! 암호화해서 저장하고, 본인만 볼 수 있어요.

**Q: 음질이 안 좋아도 되나요?**
A: 되도록 조용한 곳에서 녹음하면 좋아요.
   너무 시끄러우면 AI가 잘 못 알아들어요.

**Q: 여러 명이 동시에 말하면?**
A: AI가 최대한 구분하려고 하지만,
   한 명씩 말하는 게 가장 정확해요.
