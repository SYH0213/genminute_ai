## 1️⃣ 보안 기본 질문

### Q76. 내 자료는 안전한가요?

**쉬운 답변**:
네! 여러 겹으로 보호하고 있어요.

**보안 장치 5개**:

**1. 로그인 (1단계)**
```
구글 계정으로만 로그인
→ 구글이 보안 관리 (세계 최고 수준!)
```

**2. 파일 암호화 (2단계)**
```
원본: "1차회의.mp3"
저장: "a3f7b2c1-4d5e-6f7g.mp3"
→ 파일명만 봐도 뭔지 모름!
```

**3. 접근 제한 (3단계)**
```
내 파일: 나만 볼 수 있음
공유한 파일: 허락한 친구만
```

**4. 인터넷 암호화 (4단계)**
```
HTTPS (자물쇠 표시)
→ 중간에서 가로채도 읽을 수 없음
```

**5. 서버 보안 (5단계)**
```
구글 클라우드 사용
→ 해킹 방지 시스템 자동 작동
```

### Q77. 비밀번호는 어떻게 관리하나요?

**답변**:
우리는 비밀번호를 저장 안 해요!

**왜요?**
- 구글 계정 사용
- 구글이 비밀번호 관리
- 우리는 "이 사람 구글 회원 맞아요" 확인만!

**비유**:
- 놀이공원 입장
- 우리: "신분증 보여주세요"
- 구글: "신분증 맞아요!" (신분증은 구글이 관리)
- 우리: "들어오세요!"

**장점**:
- 비밀번호 유출 걱정 없음
- 안전!

### Q78. 해킹당할 수 있나요?

**답변**:
100% 안전은 없지만, 매우 어려워요!

**보안 수준**:

| 공격 방법 | 방어 수준 |
|----------|----------|
| 비밀번호 탈취 | 불가능 (우리가 저장 안 함) |
| 파일 가로채기 | 어려움 (암호화) |
| 서버 해킹 | 매우 어려움 (구글 클라우드) |
| 로그인 가장 | 어려움 (구글 인증) |

**비유**:
- 은행 금고 수준 🏦
- 집 창문보다 훨씬 안전!

### Q79. 다른 사람 파일 볼 수 있나요?

**답변**:
절대 안 돼요! 철저히 차단돼 있어요.

**시도해도**:
```
A가 B의 파일 주소를 알아냄
    ↓
A가 접속 시도
    ↓
시스템: "권한 없음! 차단!"
    ↓
A 화면: "접근 권한이 없습니다 (403)"
```

**권한 확인**:
1. 로그인 확인
2. 내 파일인지 확인
3. 공유받은 파일인지 확인
4. 모두 아니면 차단!

### Q80. 인터넷이 안전하지 않으면?

**답변**:
HTTPS로 암호화해서 안전해요!

**비유**:

**HTTP (일반)** ❌:
```
나 → 편지 → 서버
(누구나 열어볼 수 있음!)
```

**HTTPS (보안)** ✅:
```
나 → 자물쇠 편지 → 서버
(자물쇠 열쇠는 나와 서버만 가짐!)
```

**확인 방법**:
- 주소창에 🔒 자물쇠 표시
- "https://" 로 시작
